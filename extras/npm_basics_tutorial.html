NPM Basics - A Beginner's Guide

Verifying NPM Version and Updating NPM

To check what version of npm (Node Package Manager) is installed on your system, you can use the following command in your terminal: 

npm -v

This command will display the installed version of npm. If you want to update npm to the latest version, you can run the following command:

npm install -g npm

This will install the latest version of npm globally on your system.

Initializing a New Package or Project with NPM

To initialize a new project using npm, navigate to your project directory and run the command: 

npm init

This will start a guided process where npm will ask you several questions about your project (such as its name, version, description, etc.) and generate a package.json file based on your input.

If you want to skip the interactive prompt and create the package.json file with default values, you can use the following command:

npm init -y

The -y flag automatically confirms all the default values.

Using the NPM Registry to Find a Package

The npm registry is a large collection of packages published by developers around the world. To find a specific package, you can visit the official npm website at npmjs.com and search for the package by name.

Alternatively, you can use npm’s search feature from the command line to find packages:

npm search

This command will list all packages that match your search term, along with brief descriptions and details.

Installing a Package with NPM

Once you have identified a package you want to install, you can use npm to install it. For example, to install the popular lodash package, run the following command: 

npm install lodash

This will install the package and add it to your project’s node_modules directory. Additionally, it will update your package.json file by adding lodash to the list of dependencies.

If you want to install a package globally so it can be used across multiple projects, you can add the -g flag:

npm install -g lodash

Using an NPM Package in Your Code

After installing a package, you can use it in your code by requiring or importing it, depending on the module system your project is using. In Node.js, you can use the require() function to import the package. 


// Using lodash in Node.js
const _ = require('lodash');

const arr = [1, 2, 3];
const reversedArr = _.reverse(arr);
console.log(reversedArr);  // Output: [3, 2, 1]

In ES6 module syntax, you would use the import statement instead:

import _ from 'lodash';

Dependencies vs Development Dependencies

In npm, dependencies are packages that your project needs in order to function. These are packages required at runtime and are typically installed with the following command: 

npm install

Development dependencies, on the other hand, are packages that are only needed during development (such as testing frameworks or build tools). These dependencies are installed with the --save-dev flag: 

npm install  --save-dev

For example, you might install jest (a testing framework) as a development dependency:

npm install jest --save-dev

The key difference is that development dependencies are only used during the development phase, while regular dependencies are needed for the application to run in production.
