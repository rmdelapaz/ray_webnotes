Advanced NPM Usage - A Beginner's Guide

1. Installing an Existing Project's Dependencies

When you clone or download a project that has a package.json file, you will need to install the project’s dependencies to get it running. This is done using the following command: 

npm install

This command reads the package.json file and installs all the listed dependencies into the node_modules directory. It ensures that the project has all the necessary packages to work as expected.

2. Uninstalling a Package with NPM

If you no longer need a package in your project, you can uninstall it using the following command: 

npm uninstall

This removes the package from your project’s node_modules folder and updates the package.json file to reflect the change. For example, to uninstall lodash, you would run:

npm uninstall lodash

3. Updating a Package with NPM

Packages can have newer versions released over time, and you may want to update them in your project to get new features or security patches. To update a specific package to its latest version, you can run: 

npm update

For example, to update the express package, you would run:

npm update express

If you want to update all the packages in your project to their latest compatible versions, simply run:

npm update

4. Finding and Fixing NPM Package Security Vulnerabilities

Security vulnerabilities can arise in the packages your project depends on. NPM provides a built-in security audit tool to help you identify and fix these vulnerabilities. To run a security audit, use the following command: 

npm audit

This command will scan your project’s dependencies and list any security vulnerabilities it finds. To automatically fix the vulnerabilities, you can run:

npm audit fix

This will attempt to resolve security issues by upgrading vulnerable packages to safer versions.

5. Writing and Running NPM Scripts

NPM allows you to define custom scripts that can be executed from the command line. These scripts are defined in the scripts section of the package.json file. For example, you can add a script to start your application or run tests. 


{
  "scripts": {
    "start": "node app.js",
    "test": "jest"
  }
}

Once the scripts are defined, you can run them using the following command:

npm run

For example, to run the start script, you would use:

npm run start

If the script name is one of npm's default script names like start or test, you can omit the run keyword and simply use:

npm start
