JavaScript's Event Loop Model

JavaScript uses an event loop model to handle asynchronous operations, allowing the language to perform non-blocking tasks despite being single-threaded.

Metaphor: Think of JavaScript like a chef who can only cook one dish at a time (single-threaded). However, the chef has a system to handle multiple orders (tasks) by checking for new orders when they are done with the current dish. This is the event loop at work, managing tasks asynchronously while keeping the chef focused on one task at a time.

JavaScript as a Single-Threaded Language

JavaScript is single-threaded, meaning it can only execute one task at a time. However, using asynchronous code and the event loop, JavaScript can handle tasks that don't block the main thread, such as fetching data from a server or waiting for user input.

Synchronous vs. Asynchronous Code

Synchronous code is executed line by line, where each operation must complete before the next one can begin. Asynchronous code, on the other hand, allows the program to continue executing without waiting for an operation to complete.

// Example of synchronous code
console.log("Start");
console.log("End");
// Output:
// Start
// End

// Example of asynchronous code with setTimeout
console.log("Start");
setTimeout(() => {
    console.log("Async operation finished");
}, 1000);
console.log("End");
// Output:
// Start
// End
// Async operation finished (after 1 second)

Executing the Asynchronous Function setTimeout

setTimeout is a built-in function in JavaScript that allows you to execute a function after a specified delay. The function provided as the first argument is called a "callback." Here's an example:

setTimeout(() => {
    console.log("This message will appear after 2 seconds");
}, 2000);

This code will print the message after a delay of 2 seconds.

Predicting the Output of the asyncy Function

Given the function function asyncy(cb) { setTimeout(cb, 1000); console.log("async"); } and the callback function callback() { console.log("callback"); }, let's predict the output:

function asyncy(cb) {
    setTimeout(cb, 1000);
    console.log("async");
}

function callback() {
    console.log("callback");
}

asyncy(callback);

// Output:
// async
// callback (after 1 second)

The function asyncy logs "async" first because setTimeout is asynchronous. The callback is then executed after 1 second, printing "callback."

Using setInterval to Execute a Function 10 Times

setInterval is similar to setTimeout, but it repeatedly executes a function at a specified interval until it is cleared. Here's how you can use it to execute a function 10 times with a 1-second interval:

let count = 0;
const intervalId = setInterval(() => {
    console.log("Interval function", count + 1);
    count++;
    if (count === 10) {
        clearInterval(intervalId);
        console.log("Interval cleared");
    }
}, 1000);

This code will print the message every second, incrementing the count each time. After 10 executions, it will clear the interval.

Using Node's Readline Module to Accept User Input

In Node.js, you can use the readline module to accept input from the user. Here's an example:

const readline = require('readline');

const rl = readline.createInterface({
    input: process.stdin,
    output: process.stdout
});

rl.question("What is your name? ", (answer) => {
    console.log(`Hello, ${answer}!`);
    rl.close();
});

When you run this program, it will ask for the user's name, and after the input is provided, it will greet the user.
