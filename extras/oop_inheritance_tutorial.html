OOP Inheritance in JavaScript - A Beginner's Guide

What is Inheritance in OOP?

Inheritance is a fundamental concept of Object-Oriented Programming (OOP). It allows one class (called a subclass or derived class) to inherit properties and methods from another class (called a superclass or base class).

Analogy: Imagine inheritance like passing down family traits. For example, a child may inherit physical traits like eye color or skills from their parents. In programming, a subclass inherits characteristics from its superclass, and it can also have its own unique traits.

In JavaScript, we use the extends keyword to make one class inherit from another. Let’s dive into how this works.

Why Use Inheritance?

Inheritance is useful because it allows us to reuse code, reduce duplication, and create more structured, maintainable programs. Instead of rewriting similar code in multiple places, we can centralize common functionality in a base class and extend it when necessary.

Real-world example: Consider an application for managing vehicles. You may have different types of vehicles, like cars, trucks, and motorcycles. All vehicles share common characteristics (such as make, model, and year), so you can define those in a parent Vehicle class. Each specific type of vehicle can then extend the Vehicle class to inherit its properties and methods.

Defining a Superclass and Subclass

To demonstrate inheritance, let’s define a basic superclass, Vehicle, and then extend it to create a subclass, Car.


class Vehicle {
    constructor(make, model, year) {
        this.make = make;
        this.model = model;
        this.year = year;
    }

    start() {
        console.log(`${this.make} ${this.model} is starting.`);
    }
}

class Car extends Vehicle {
    constructor(make, model, year, doors) {
        super(make, model, year);
        this.doors = doors; // unique property for Car
    }

    honk() {
        console.log(`${this.make} ${this.model} is honking its horn!`);
    }
}

const myCar = new Car('Toyota', 'Camry', 2021, 4);
myCar.start(); // Inherited from Vehicle class
myCar.honk();  // Defined in Car class

Here, Car extends Vehicle, meaning it inherits the properties and methods from Vehicle. The super() function in the constructor is used to call the constructor of the superclass, allowing us to reuse the code from Vehicle without duplicating it.

Practical Uses of Inheritance

Inheritance is widely used in real-world applications to create more flexible and modular code. For example, in game development, you might have a superclass Character with common properties like health and level. You could then create subclasses like Warrior and Mage that extend Character, each with their own specialized abilities.

Real-world analogy: Consider a company hierarchy. At the top, you have a base "Employee" class that represents the general characteristics of an employee (like name and ID). From this base class, you can have specialized subclasses like "Manager" and "Developer", which inherit common characteristics but also have their own specific roles and responsibilities.

Overriding Methods in the Subclass

Sometimes, a subclass might need to modify or replace a method from its superclass. This is called method overriding. You can define a method in the subclass with the same name as a method in the superclass, and it will override the original method. 


class Car extends Vehicle {
    constructor(make, model, year, doors) {
        super(make, model, year);
        this.doors = doors;
    }

    // Overriding the start method
    start() {
        console.log(`${this.make} ${this.model} is revving its engine.`);
    }
}

const myCar = new Car('Ford', 'Mustang', 2020, 2);
myCar.start(); // Ford Mustang is revving its engine.

Here, we have overridden the start() method in the Car class to provide custom functionality specific to cars.

Polymorphism in Action

In OOP, polymorphism means "many forms." Inheritance allows objects to be treated as instances of their parent class, enabling more flexible and dynamic code.

Example: Imagine a function that works with any type of vehicle. Thanks to inheritance, we can pass in different vehicle types (like Car or Motorcycle), and the function can call the same method on them even though they are different types of objects.


function testStart(vehicle) {
    vehicle.start();
}

const myCar = new Car('Honda', 'Civic', 2018, 4);
const myVehicle = new Vehicle('Generic', 'Model', 2015);

testStart(myCar);       // Honda Civic is revving its engine. (from Car)
testStart(myVehicle);   // Generic Model is starting. (from Vehicle)

This demonstrates polymorphism, where the testStart() function works with both Car and Vehicle instances.

Common Bugs and How to Avoid Them

When working with inheritance, here are some common issues to be aware of:

  • Not calling super() in the subclass constructor: Forgetting to call super() will result in an error because the base class constructor needs to be initialized before the subclass can add its own properties.
  • Overriding methods without using the same method signature: Ensure that when you override methods, you keep the method signature (parameters) the same, or else you might break the behavior of the inherited method.
  • Accidental method overrides: Be careful to use unique method names unless you specifically intend to override a method.
