Automated Testing: The How, What, and Why

The What: What Is Automated Testing?

Automated testing allows developers to write code that tests their application code. These tests ensure that a piece of code behaves as expected under specific conditions. Instead of manually checking if each part of your program works, you automate this process using testing frameworks.

Example:

Let’s say you’ve written a function that adds two numbers. An automated test would look like this: 

function add(a, b) {
    return a + b;
}

// Test
console.assert(add(2, 3) === 5, "Test failed: 2 + 3 should equal 5");

This test verifies that add(2, 3) produces 5. If the result is different, the test fails and outputs the failure message.

The How: How Do We Test?

Testing typically follows three steps:

  1. Set up a specific condition.
  2. Execute the code.
  3. Verify the result matches the expected behavior.

Many testing frameworks follow this pattern, often called Arrange, Act, Assert.

Example:

Let’s revisit the earlier example using a testing framework syntax: 

describe('add', function() {
    it('should return 5 when adding 2 and 3', function() {
        const result = add(2, 3);
        assert.equal(result, 5);
    });
});

Here, the describe and it blocks arrange the test case, the add(2, 3) function call is the action, and assert.equal(result, 5) verifies the outcome.

The Why: Why Do We Test?

The main goal of testing is to give developers confidence that their code works as expected. Without automated tests, it’s easy for bugs to slip through, especially as applications grow more complex. Testing also allows you to make changes to your code (refactoring) with less fear of breaking existing functionality.

Benefits of Automated Testing:

  • Early Bug Detection: Catch errors before they make it to production.
  • Refactoring Safety: Make changes confidently, knowing tests will catch issues.
  • Documentation: Tests serve as a form of documentation, showing how functions or modules are expected to behave.
  • Faster Development: Running tests frequently helps you identify and fix issues faster, saving time in the long run.

Reading Automated Tests

You don't need to understand every detail of a testing framework’s syntax to understand what a test is doing. Focus on these three key points:

  1. What is being tested? Identify the function or module being tested.
  2. What are the inputs? What values are being passed into the function?
  3. What is the expected outcome? What result should the test expect?

Example Breakdown:

describe('multiply', function() {
    it('should return 6 when multiplying 2 and 3', function() {
        const result = multiply(2, 3);
        assert.equal(result, 6);
    });
});

- What is being tested? The multiply function.
- What are the inputs? 2 and 3.
- What is the expected outcome? The result should be 6.

Conclusion

Automated testing is an essential practice in modern software development. By writing tests, you ensure that your code behaves as expected and reduces the risk of bugs making it into production. It’s also a great tool for improving code quality and maintaining confidence as your application evolves.
