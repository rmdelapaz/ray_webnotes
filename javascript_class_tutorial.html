Understanding Classes in JavaScript (ES6) - A Beginner's Guide

1. Defining a Class using ES6 Syntax with a Constructor Method

In ES6, defining a class is much simpler compared to the older constructor functions in JavaScript. The keyword class is used to declare a class, and the constructor method is used to initialize the instance variables when an object is created.

Analogy: Think of a class like a blueprint for a house. It defines what every house (object) built from the blueprint should have (instance variables like rooms, doors) and what the house can do (methods like open doors, ring the bell).


class Car {
    constructor(make, model, year) {
        this.make = make;
        this.model = model;
        this.year = year;
    }
}

In this example, the Car class has a constructor that takes three parameters: make, model, and year. When a new instance of a Car is created, these values are assigned to the instance.

2. Defining Instance Methods and Variables

Instance methods are functions that belong to the instances of a class. Variables that belong to these instances are called instance variables. Every time you create a new object, it gets its own set of these variables and methods.

Real-world analogy: Think of instance methods as the unique abilities of each object. For example, if each car (object) has the ability to start or stop, those are the instance methods that only the car object can perform.


class Car {
    constructor(make, model, year) {
        this.make = make;
        this.model = model;
        this.year = year;
        this.isRunning = false; // An instance variable
    }

    start() {
        this.isRunning = true;
        console.log(`${this.make} ${this.model} is now running.`);
    }

    stop() {
        this.isRunning = false;
        console.log(`${this.make} ${this.model} has stopped.`);
    }
}

Here, we defined two instance methods: start() and stop(), which control whether the car is running.

3. Creating an Instance using the new Keyword

To create an object from a class, we use the new keyword followed by the class name and any required arguments for the constructor. 


const myCar = new Car('Toyota', 'Camry', 2021);
myCar.start(); // Toyota Camry is now running.

This creates a new instance of the Car class called myCar, and it can now use the methods we defined in the class, like start() and stop().

4. What are Instance Methods?

Instance methods are functions defined inside a class that operate on the individual instance of the object. These methods can modify the instance's properties and interact with the object.

Analogy: Think of instance methods as unique actions a particular object can perform, like how a particular car can start or stop. While every car may have the capability to start and stop, each car’s specific behavior when starting is tied to its own state (e.g., whether the engine is already on).

5. How to Call Instance Methods on an Instance

To call an instance method, simply use dot notation on the instance of the class. 


myCar.stop(); // Toyota Camry has stopped.

This will call the stop() method on the myCar instance, which in turn will change the isRunning variable to false.

6. Checking if an Object is an Instance of a Class Using the instanceof Operator

The instanceof operator checks whether a specific object was created using a particular class. This is useful for debugging or verifying the type of object you're working with. 


console.log(myCar instanceof Car); // true

In this case, the expression returns true because myCar was created using the Car class.

7. Debugging Common Bugs When Using Classes in JavaScript

Here are some common issues you may run into when working with classes and how to resolve them:

  • Uncaught TypeError: Cannot read property '...' of undefined: This happens if you try to access a property that hasn’t been initialized yet. Double-check that your constructor is correctly assigning instance variables.
  • Missing new when creating an instance: Forgetting to use the new keyword will cause JavaScript to treat the class like a regular function, leading to unexpected behavior.
  • Using arrow functions for instance methods: Avoid using arrow functions for methods that need access to this, as arrow functions bind this lexically, which may not refer to the correct instance.
