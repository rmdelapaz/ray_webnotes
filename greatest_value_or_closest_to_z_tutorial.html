Greatest Value or Closest to 'Z' Function - Using Reduce

Applying George Polya's 4-Step Problem-Solving Method

Step 1: Understand the Problem

The challenge is to create a function that takes an array of numbers or letters and returns the element that has the greatest value or the letter closest to 'z' if itâ€™s a string. We will use the JavaScript reduce() method to solve this problem.

Expected Input and Output:

  • findGreatest([3, 5, 2, 9]) should return 9.
  • findGreatest(['a', 'x', 'b', 'z']) should return 'z'.
  • findGreatest([5, 'x', 2, 'y']) should return 'y'.

Step 2: Devise a Plan (Pseudocode/Whiteboard)

Let's break down the steps needed to solve the problem:


// Pseudocode for finding the greatest value using reduce()
function findGreatest(arr) {
    return arr.reduce((max, current) => {
        // If the current value is greater than max, update max
        // If both are letters, compare based on their Unicode value
        // If it's a mix of numbers and letters, treat letters as greater than numbers
        return current is greater than max ? current : max
    }, arr[0]); // Initialize the max value to the first element of the array
}

Step 3: Carry Out the Plan (Basic JavaScript Solution)


// Basic solution using reduce() to find the greatest value
function findGreatest(arr) {
    return arr.reduce((max, current) => {
        // Compare numbers and letters
        if (typeof current === 'string' && typeof max === 'number') {
            return current;
        } else if (typeof current === 'number' && typeof max === 'string') {
            return max;
        }
        return current > max ? current : max;
    }, arr[0]);
}

// Example usage:
console.log(findGreatest([3, 5, 2, 9])); // 9
console.log(findGreatest(['a', 'x', 'b', 'z'])); // 'z'
console.log(findGreatest([5, 'x', 2, 'y'])); // 'y'

In this basic solution, we use the reduce() method to iterate through the array, comparing each element to the current maximum. If the current value is greater than the maximum, it becomes the new maximum. For mixed arrays of numbers and letters, we treat letters as greater than numbers.

Step 4: Review and Improve (Advanced Solution)

The basic solution works well, but we can make improvements. For example, we could handle case sensitivity by converting letters to lowercase before comparison. This way, 'Z' and 'z' would be treated equally. We can also refine how we compare mixed data types (numbers and letters).


// Advanced solution with case-insensitive comparison
function findGreatest(arr) {
    return arr.reduce((max, current) => {
        // Handle case-insensitivity by converting both to lowercase if they are strings
        const currentVal = typeof current === 'string' ? current.toLowerCase() : current;
        const maxVal = typeof max === 'string' ? max.toLowerCase() : max;

        // Compare numbers and letters, handling mixed types
        if (typeof currentVal === 'string' && typeof maxVal === 'number') {
            return current;
        } else if (typeof currentVal === 'number' && typeof maxVal === 'string') {
            return max;
        }
        return currentVal > maxVal ? current : max;
    }, arr[0]);
}

// Example usage:
console.log(findGreatest([3, 5, 2, 9])); // 9
console.log(findGreatest(['a', 'x', 'b', 'z'])); // 'z'
console.log(findGreatest(['A', 'x', 'B', 'Z'])); // 'Z'
console.log(findGreatest([5, 'x', 2, 'y'])); // 'y'
